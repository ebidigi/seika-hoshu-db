# 成果報酬チーム管理DB - 要件定義書

## 1. システム概要

成果報酬チーム専用の営業管理ダッシュボード（SPA）。
スプレッドシートの実績データをGAS経由でTurso DBに同期し、ブラウザからリアルタイムに進捗を可視化する。

## 2. データフロー

```
スプレッドシート（実績rawdata / 売上報告rawdata）
  ↓ GAS定期同期（SyncToTurso.js）
  ↓ normalizeMemberName() でメンバー名を正規化
Turso DB（seika-hoshu-db）
  ↓ フロントエンド直接クエリ（app.js → Turso HTTP API）
ダッシュボード（index.html）
```

## 5. GAS機能

### 5.1 データ同期（SyncToTurso.js）

- `syncPerformanceToTursoSeika()`: 実績rawdata → performance_rawdata テーブル
- `syncSalesReportToTursoSeika()`: 売上報告rawdata → appointments テーブル + projects自動登録
- フィルタ: `isSeikaTeamMember()` で成果報酬チームメンバーのみ対象
- 名前正規化: `normalizeMemberName()` でSlack形式→DB正規名に変換

### 5.2 Slack通知（SlackNotify.js）

- チャンネル: `C0ACA4Q05PB`
- 定時: 12:00, 15:00, 18:00, 20:00
- 内容: 全体進捗 + チーム別内訳 + 未確認アポ警告

### 5.3 APIエンドポイント（Code.js）

- `?action=sync`: 全データ同期実行
- `?action=status`: 同期状態確認

## 7. ビジネスロジック

- **売上 = アポ単価 × アポ件数 × 実施率（≒80%）**
- キャンセル → 請求¥0
- リスケ → 翌月流れ（≒50%）
- **収益性アラート**: 単価 × 架電toアポ率 < 7.5 → 警告
- **月間目標**: ¥9,000,000（全体デフォルト）
- **営業日計算**: 土日祝（holidaysテーブル）を除外
