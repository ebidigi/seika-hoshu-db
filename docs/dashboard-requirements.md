# CROダッシュボード（dashboard.html）要件定義書

## 1. 概要

**プロダクト名**: DigiMan チーム900 CROダッシュボード
**対象ユーザー**: チーム900 マネージャー・メンバー
**目的**: 成果報酬チーム（菊池Team・三善Team）と稼働報酬Teamの営業KPI・売上進捗・歩留まりを一元管理し、リアルタイムで可視化する

---

## 2. システム構成

| 項目 | 内容 |
|------|------|
| フロントエンド | HTML + CSS + Vanilla JS（SPA構成） |
| データベース | Turso（libSQL）— 成果報酬チーム用 |
| 稼働報酬データ | GAS Web App API（スプレッドシート経由） |
| グラフ描画 | Chart.js + chartjs-plugin-annotation |
| デザイン | DigiManブランドカラー（Blue: #1155cc, Cyan: #00a2da, Yellow: #e8d335, Red: #e04f24） |

## 3. ページ構成（トップレベルタブ）

### 3.1 サマリ

全チーム横断のダッシュボード。

| セクション | 内容 |
|-----------|------|
| 全体KPIバー | 全体売上目標 / 取得実績 / 達成率プログレスバー / 標準進捗ライン / 営業日経過 / 実施確定金額 |
| チーム比較テーブル | 菊池Team vs 三善Team の主要指標比較（取得目標・実績・達成率・実施目標・実施確定・実施見込(0.85)・架電数・PR数・アポ数・アポ単価・各種歩留まり率・メンバー数・案件数） |
| アポステータス | 未確認 / 実施 / リスケ / キャンセル の件数内訳（チーム別 + 全体）、実施率 |
| メンバーランキング | 取得金額降順のメンバーランキング（チーム所属表示） |

#### 3.2.1 概要サブタブ

| 要素 | 内容 |
|------|------|

| メンバーカード | メンバーごとに架電・PR・アポ実績、取得金額、進捗バーを表示 |

#### 3.2.2 アポ確認サブタブ

| 要素 | 内容 |
|------|------|
| ステータスサマリバー | 全体・未確認・実施・リスケ・キャンセルの件数バッジ |

#### 3.2.3 案件管理サブタブ

| 要素 | 内容 |
|------|------|
| アサインメント追加ボタン | 新規アサインメントをモーダルで追加 |
| 案件カード | アサインされた案件ごとに、キャップ進捗・取得金額を表示 |

**アサインメントCRUD**:
- `project_member_assignments` テーブルに対する INSERT / UPDATE / DELETE
- フィールド: member_name, project_name, year_month, project_type（成果報酬/稼働報酬）, pm_name, cap_count, cap_amount, target_count, sheet_url

#### 3.2.4 歩留まり・分析サブタブ

| 要素 | 内容 |
|------|------|
| 歩留まりファネル | 架電 → PR → アポ → 実施 のファネル表示（各段の転換率） |
| メンバー別歩留まりテーブル | 架電・PR・アポ・架toPR・PRtoアポ・架toアポ・取得金額（基準値色分け） |
| 診断・改善示唆 | 歩留まり基準値との比較でアラート/示唆を自動生成 |
| 収益性アラート | 単価 × 架電toアポ率 < 7 の案件を検出（架電50件以上） |
| トレンドチャート | 架電/PR/アポ/金額の日次棒グラフ + 累積折れ線（Chart.js） |
| 日次データテーブル | 日付別の架電・PR・アポ・金額・歩留まり率 |

**歩留まり基準値**:
| 指標 | 基準値 | 注意 | 要改善 |
|------|--------|------|--------|
| 架電toPR | 15% | 10.5%〜15% | <10.5% |
| PRtoアポ | 30% | 21%〜30% | <21% |
| 架電toアポ | 3% | 2.1%〜3% | <2.1% |

### 3.3 稼働報酬Team

GAS Web App APIからデータ取得。LocalStorageでキャッシュ（5分TTL）。

#### 3.3.1 概要サブタブ

| 要素 | 内容 |
|------|------|
| 売上サマリー | 確定売上・達成率プログレスバー・目標・差分・見込（パイプライン込） |
| 売上詳細テーブル（トグル） | 四半期内月別の目標金額・確定案件・パイプライン案件・合計・差分 |
| KPI進捗カード | 標準進捗 / 架電進捗 / アポ進捗 / 進捗差分（前日/本日/明日切替） |
| フィルター | 案件フィルター + 担当者フィルター（トグルボタン） |
| 担当者別進捗カード | 架電/アポ進捗バー（標準進捗ライン表示）・実績/目標・偏差表示 |
| チャート | 個人別架電vsアポ進捗（棒グラフ）/ 架電進捗ランキング / アポ進捗ランキング |

**進捗モード切替**:
- 前日 / 本日 / 明日の3モード
- 営業日ベースで標準進捗を計算（土日祝除外）

#### 3.3.2 分析サブタブ

| 要素 | 内容 |
|------|------|

| 実数指標 | 架電数・PR数・アポ数・架電数/H |
| 率指標 | 架電toPR率・PRtoアポ率・架電toアポ率（先月比・通算比表示） |
| 日次推移チャート | 7種のチャートタイプ切替（架電数/PR数/アポ数/架toPR率/PRtoアポ率/架toアポ率/架電数H） |

#### 3.3.3 歩留まりサブタブ

| 要素 | 内容 |
|------|------|
| 歩留まりファネル | 架電 → PR → アポ のファネル |
| メンバー別歩留まりテーブル | 基準値色分け（成果報酬チームと同一基準） |
| 診断・改善示唆 | 自動生成のアラート/改善提案 |

#### 3.3.4 設定サブタブ

| 要素 | 内容 |
|------|------|
| 月別売上目標設定 | 当四半期の月ごとに売上目標金額を入力・保存 |

**案件CRUD（稼働報酬）**:
- GAS Web App経由で案件の追加・編集・削除
- フィールド: deal_name, owner, phase（提案前/提案済み/見積もり提出済み/受注/保留/失注）, amount, probability, expected_start_date, next_action, action_deadline, memo

## 7. ビジネスロジック

### 7.1 標準進捗

```
標準進捗(%) = 経過営業日 / 全営業日 × 100
```
- 営業日 = 平日（月〜金） かつ 祝日でない日
- 祝日: `holidays` テーブル + フォールバック定数（2026年）

### 7.2 実施見込

```
実施見込 = 実施確定金額 + 未確認アポ金額 × 0.85
```

### 7.3 達成率

```
達成率(%) = 実績金額 / 目標金額 × 100
```

### 7.4 進捗ステータス色分け

| 条件 | ステータス |
|------|----------|
| 実績進捗率 ≥ 標準進捗 | 成功（青） |
| 実績進捗率 ≥ 標準進捗 × 0.8 | 警告（黄） |
| 実績進捗率 < 標準進捗 × 0.8 | 危険（赤） |

### 7.5 データ重複排除

- **アポイント**: `member_name|project_name|acquisition_date|customer_name` をキーに重複排除（confirmation_date有を優先）
- **実績**: `member_name|project_name|input_date` をキーに重複排除（updated_at新しい方を優先）

## 8. キャッシュ戦略（稼働報酬）

| 対象 | キー | TTL |
|------|-----|-----|
| 月次サマリー | `kadou_monthly_{month}` | 5分 |
| パイプライン | `kadou_pipeline` | 5分 |
| 日次生データ | `kadou_rawdata_{startDate}_{endDate}` | 5分 |

- LocalStorageに保存
- 月切替時にキャッシュクリア
- 案件保存/削除時にパイプライン・月次キャッシュをクリア
