## FBの本質整理

安達さんが求めていたのは「数字を見る」ではなく「数字で判断する」ダッシュボード。

```
現状:  データ → 表示（カード羅列）→ 人が頑張って読み解く → 判断
理想:  データ → 構造化表示 → 一目で異常がわかる → 即判断・アクション
```

| 今の問題 | 求められていること |
|---------|------------------|
| 数字が並んでいるだけ | 「やばい」がパッと見でわかる |
| 全指標が同列 | 重要な順にメリハリがある |
| ページ遷移で迷子 | 1画面で完結 or 自然な導線 |
| チーム横断で見れない | チーム比較→ドリルダウンで原因特定 |
| 作った人しかわからない | 初見でも3秒で読める |

### 2. 菊池さん（営業リーダー）に聞くこと — 見せ方アドバイス

> **前提共有**: ダッシュボードの見せ方を作り直す。実際に毎日使う人として、何が必要で何が不要か教えてほしい。

#### 日常の使い方

**経営会議の場で**
- [ ] 数字を報告するとき、どの指標をどの順番で話す？

- [ ] 「このチームやばい」と判断するとき、何の数字を見る？

**日常運用で**
- [ ] 朝イチで確認する情報は何？（アポ確認？昨日の実績？）
- [ ] メンバーにフィードバックするとき使う数字は？

**今のダッシュボードの問題点**
- [ ] 「ここ見づらい」「ここいらない」はどこ？
- [ ] 「これが見たいのにない/見つけにくい」はある？
- [ ] 実施目標と取得目標、どっちを主軸に見ている？

### 3. 菊池さん（動線設計）— 一緒に作業するための論点

> **前提**: 安達さんから動線設計の見直しを指示された。菊池さんとミーティングして再設計する。

#### 設計前に決めるべきこと

**ユーザーと利用シーン**
- [ ] 誰が、いつ、何のために見る？（最低3パターン洗い出す）
  - 例: 経営会議で全体進捗を報告 / 朝イチでアポ確認 / 夕方に実績振り返り
- [ ] 一番多いユースケースは何？ → それがトップページになるべき

**ページ構成案（叩き）**
```
案A: タスクベース
  トップ（全体ヘルスチェック）→ アポ確認 → 詳細分析

案B: ドリルダウン型
  全体サマリ → チーム選択 → メンバー詳細

案C: ロールベース
  マネージャービュー（判断用） / メンバービュー（自分の数字）
```
- [ ] どの方針がうちに合う？

**「判断できるダッシュボード」にするための設計**
- [ ] トップで見せるべき指標TOP5は何？
- [ ] 「やばい」の定義は？（目標対比XX%未満？標準進捗対比？）
- [ ] 異常値を見つけた後の次アクションは何？（→そこへの導線を作る）
